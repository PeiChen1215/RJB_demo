## 一、完成内容

### 1. A11：Agent 超时熔断与降级

**文件：** `backend/app/agents/orchestrator.py`

- 新增进程内 `_CircuitBreaker`：60 秒内同一 Agent 失败/超时 3 次即打开熔断器，冷却 120 秒后自动关闭。
- 新增全局 `_AGENT_EXECUTOR` 线程池，为同步 `agent.run(msg)` 增加 `timeout=10` 控制。
- 重写了 `_safe_run`（同步）与 `_safe_run_async`（异步 SSE）：
  - 超时返回 `{"fallback": true, "reason": "timeout"}`
  - 异常返回 `{"fallback": true, "reason": "exception"}`
  - 熔断开启返回 `{"fallback": true, "reason": "circuit_open"}`
- SSE 流式生成中任一 Agent 超时/熔断会立即 `yield {"type": "error", ...}` 并结束生成；`app/api/resources.py` 在 `final_result` 为空时把 `generation_task` 标记为 `failed`。

### 2. A10：辩论议会与知识熔炉联动

**新增文件：**

- `backend/app/services/knowledge_furnace.py`
- 测试：`backend/tests/test_knowledge_furnace.py`

实现逻辑：

1. 判题接口 `/api/code/judge` 与 `/api/code/judge-exercise` 在学生提交失败且带 `concept` 时，通过 `BackgroundTasks` 后台调用 `trigger_resource_review(concept)`。
2. `should_trigger_resource_review` 判断条件：
   - 该知识点总提交数 `>= 5`
   - 错误率 `>= 0.6`
3. 触发后：
   - 创建新的 `generation_task`（status = pending）
   - 调用 `AgentOrchestrator.generate_resource` 重新生成资源并执行辩论议会
   - 将新内容 upsert 到 `resource` 表，版本号 +1
   - 通过 `save_resource_version()` 写入 `resource_version` 演进记录
   - 任务完成后 `generation_task.status = completed`
4. 返回结果增加字段：
   - `knowledge_furnace_triggered: true/false`
   - `concept`
