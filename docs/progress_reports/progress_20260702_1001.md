### 1. C4：Demo 主链路跑通

**新增脚本：** `logs/c4_demo_chain.py`

按“登录 → 画像 → 路径 → 资源 → 辩论 → 代码 → 辅导 → 报告”逐节点调用：

- `POST /api/sessions/` 创建会话并返回画像/推荐路径；
- `POST /api/sessions/{id}/chat` 触发“变量与赋值”学习资源生成（Navigator → Generator → Reviewer）；
- `POST /api/code/judge` 提交正确代码并通过判题；
- `POST /api/sessions/{id}/chat` 模拟代码报错，进入苏格拉底辅导；
- `POST /api/sessions/{id}/evaluate` 生成学习评估；
- `GET /api/sessions/{id}/stats` 拉取学习统计。

**结果：** Mock 模式下整条链路跑通，无 500，各环节返回结构符合预期。

### 2. A9：苏格拉底辅导链深度持久化

**文件：** `backend/app/agents/orchestrator.py`

- `_route` 已将会话 `session` 透传至 `_tutor_flow`；
- `_tutor_flow` 从 `session["socratic_depth"]` 读取当前深度，生成问题后写回 `session`，并在 `convergence` 后归零；
- `_is_continue_tutor` 识别“继续/继续引导/请继续/下一步/接着问”等意图，在深度 > 0 时继续走辅导流，避免回退到 Profiler；
- 辅导返回 payload 中同时包含 `message` 与 `question` 字段，兼容前端 `SocraticPanel` 的 `question` 读取。

**验证：**

- Mock 模式：第一轮代码报错返回 `agent_name=Socrates, response_type=tutor`；点击“继续引导”后仍返回 `tutor`，未回退 Profiler。
- 真实 DeepSeek 模式（直接调用 `SocratesTutor`）：`depth=0~4` 分别对应 `clarification → assumption_probe → evidence_check → counter_example → convergence`，阶段递进正确。

### 3. B10：画像证据后端聚合 + 前端展示

**后端文件：** `backend/app/api/sessions.py`

- `POST /api/sessions/{id}/behavior` 原只写入 `learning_events`；
- 现改为调用 `database.log_behavior_event`，同步生成 `cognitive_profile_evidence` 记录；
- `GET /api/sessions/{id}/profile/evidence` 按维度聚合返回最近 5 条证据。

**前端文件：** `frontend/src/App.tsx`

- `ProfileCard` 新增画像证据面板：
  - 挂载时根据 `session_id` 拉取证据；
  - 依赖 `stats.chat_count` 变化重新拉取，保证聊天/行为后实时更新；
  - 按维度折叠展示，最多显示每个维度 3 条证据。

**验证：**

- 通过行为埋点产生 `mindmap_clicked / hint_expanded / page_stay / profile_viewed` 后，证据接口返回 4 个维度数据；
- 前端 `npm run build` 通过。
