> 后端：FastAPI + Python 3.11/3.13  
> 前端：React 18 + Vite + TypeScript + Axios  
> 最后更新：2026-07-04  
> 维护人：A（后端）/ C（前端）

## 1. 文档定位

本文档是前后端的**唯一接口契约**，覆盖：

1. **已实现的全部接口**：路径、方法、请求体、响应体、状态码、异常说明。
2. **部分实现的接口**：当前可用但存在 TODO 或降级的接口，标注限制与后续计划。
3. **规划中的接口**：后续版本需要补齐的接口，供前后端提前对齐字段与交互。

任何一方修改接口字段、新增路径、调整返回结构，**必须同步更新本文档**。

## 2. 通用约定

### 2.1 Base URL

```text
开发环境：http://127.0.0.1:8000
API 前缀：/api
SSE / 静态资源：/api/xxx 或 /xxx（按接口定义）
```

前端 Axios 配置 `baseURL: '/api'`，Vite dev server 代理到后端。

### 2.2 命名规范

| 项目 | 规范 | 示例 |
|------|------|------|
| URL 路径 | 小写、连字符分隔 | `/chat-stream`、`/stream-generate` |
| 查询参数 | `snake_case` | `from_concepts`、`target_concept` |
| JSON 字段 | `snake_case` | `mastery_probability`、`is_default` |
| 前端 TypeScript | 与后端保持一致，使用 `snake_case` | `session_id`、`path_nodes` |

### 2.3 响应结构

本项目**不采用统一外层包装**（无 `code / data / message`），直接返回业务对象。

**成功示例：**

```json
{
  "session_id": "a1b2-c3d4",
  "target_concept": "变量与赋值",
  "suggested_path": ["Python简介", "变量与赋值"]
}
```

**失败示例：**

```json
// 业务错误（HTTP 200/400/404）
{ "error": "会话不存在" }
{ "success": false, "error": "未指定知识点" }

// FastAPI 校验错误（HTTP 422）
{
  "detail": [
    { "loc": ["query", "session_id"], "msg": "field required", "type": "value_error.missing" }
  ]
}
```

### 2.4 认证

- 匿名会话：无需 Token，可直接调用 `/api/sessions/`、`/api/graph/*`、`/api/code/*` 等。
- 登录用户：请求头携带 `Authorization: Bearer <jwt_token>`。
- 当前 `/api/sessions/` 列表页依赖登录，其余接口对匿名会话兼容。

### 2.5 状态图例

| 图例 | 含义 | 说明 |
|------|------|------|
| ✅ | 已实现 | 前后端已对接或后端已稳定暴露 |
| 🚧 | 部分实现 | 可用，但存在 TODO、降级、字段未持久化等问题 |
| ⏳ | 规划中 | 后端尚未实现，后续需要补齐 |

### 4.2 认证

#### ✅ POST `/auth/register`

**说明：** 用户注册，成功后自动签发 JWT。

**请求体：**

```json
{
  "username": "student01",
  "password": "123456"
}
```

**响应：**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "username": "student01"
}
```

**异常：** 用户名已存在、密码不足 6 位 → 400。

#### ✅ POST `/auth/login`

**说明：** 用户登录，使用 OAuth2 表单格式。

**请求体（`application/x-www-form-urlencoded`）：**

```text
username=student01&password=123456
```

**响应：** 同 `/auth/register`。

**异常：** 用户名或密码错误 → 401。

`chat`、`resource_generated`、`exercise_submitted`、`code_executed`、`mindmap_clicked`、`hint_expanded`、`resource_switched`、`page_stay`、`profile_viewed`、`path_viewed`、`debate_viewed`、`exercise_attempt`、`code_case_viewed`、`audio_played`、`help_requested`。

  "profile_update": { ... },

**🚧 已知限制：** 当前意图分类为关键词匹配，后续将替换为 LLM 分类。

### 4.4 知识图谱（Graph）

#### ✅ GET `/graph/`

**说明：** 返回完整知识图谱节点与前置依赖边。

**响应：**

```json
{
  "nodes": [
    { "id": "变量与赋值", "name": "变量与赋值", "module": "基础语法", "difficulty": 1 }
  ],
  "edges": [
    { "source": "Python简介", "target": "变量与赋值", "strength": 0.8 }
  ]
}
```

#### ✅ GET `/graph/layout`

**说明：** 后端驱动布局，直接供前端渲染。

**响应：**

```json
{
  "nodes": [
    {
      "id": "变量与赋值",
      "name": "变量与赋值",
      "module": "基础语法",
      "difficulty": 1,
      "x": 220,
      "y": 0,
      "color": "#3b82f6"
    }
  ],
  "edges": [
    { "source": "Python简介", "target": "变量与赋值", "strength": 0.8 }
  ]
}
```

**约定：**

- `x`、`y` 由后端基于拓扑层级计算，前端直接渲染。
- `color` 按模块分配，同模块颜色一致。

#### ✅ GET `/graph/path`

**说明：** 学习路径查询，支持三种模式：

1. **旧模式（纯路径）**

   ```text
   GET /api/graph/path?from_concepts=Python简介,基本数据类型&to_concept=变量与赋值
   ```

   **响应：**

   ```json
   {
     "from": ["Python简介", "基本数据类型"],
     "to": "变量与赋值",
     "path": ["Python简介", "变量与赋值"]
   }
   ```

2. **个人路径高亮（session 驱动）**

   ```text
   GET /api/graph/path?session_id=<uuid>&target_concept=变量与赋值
   ```

   `target_concept` 可选，默认使用会话目标。

   **响应：**

   ```json
   {
     "session_id": "uuid",
     "target_concept": "变量与赋值",
     "path_nodes": [
       {
         "id": "Python简介",
         "name": "Python简介",
         "mastery_probability": 0.95,
         "is_mastered": true,
         "is_current": false,
         "state": "mastered"
       },
       {
         "id": "变量与赋值",
         "name": "变量与赋值",
         "mastery_probability": 0.42,
         "is_mastered": false,
         "is_current": true,
         "state": "current"
       }
     ],
     "path_edges": [
       {
         "source": "Python简介",
         "target": "变量与赋值",
         "reason": "...",
         "prerequisites": [...],
         "pitfalls": [...]
       }
     ],
     "mastered_concepts": ["Python简介"]
   }
   ```

   `state` 枚举：`mastered` / `current` / `waiting`。

3. **空参数**

   **响应：** `{ "error": "请提供 from_concepts + to_concept，或 session_id + target_concept" }`（HTTP 200）。

#### ✅ GET `/graph/concept/{name}`

**说明：** 获取知识点详情。

**响应：** 知识点对象（含 `prerequisites`、`pitfalls`、`reason` 等）。

### 4.6 资源生成与知识熔炉（Resources）

#### ✅ POST `/resources/generate`

**说明：** 同步生成某个知识点的资源并执行辩论议会。

**兼容两种调用方式：**

1. 旧 query 参数：`POST /resources/generate?concept=变量与赋值`
2. JSON body：

   ```json
   {
     "concept": "变量与赋值",
     "session_id": "uuid",
     "profile": {
       "knowledge_level": 2,
       "cognitive_modality": "visual"
     }
   }
   ```

**响应：**

```json
{
  "concept": "变量与赋值",
  "package": {
    "document": "# 变量与赋值...",
    "mindmap": "...",
    "exercises": [...],
    "code_cases": [...],
    "audio_text": "..."
  },
  "debate_report": {
    "status": "PASSED",
    "rounds": [...],
    "final_votes": { "expert": "PASS", "teacher": "PASS", "student": "WARN", "guardian": "PASS" }
  }
}
```

**约定：**

- `concept` 必填。
- `session_id` 可选；提供则复用/创建该会话，否则使用临时 `default` 会话。
- `profile` 可选，仅合并到本次生成使用的临时画像中，不会覆盖原会话画像。

## 6. 数据模型字典

### 6.1 StudentProfile（学生画像）

```json
{
  "knowledge_level": 1.0,
  "cognitive_field": "dependent",
  "cognitive_modality": "visual",
  "learning_pace": "normal",
  "goal_orientation": "application",
  "error_patterns": [],
  "mastered_concepts": ["Python简介"]
}
```

- `knowledge_level`：1-5
- `cognitive_field`：`dependent` / `independent`
- `cognitive_modality`：`visual` / `auditory` / `kinesthetic`
- `learning_pace`：`slow` / `normal` / `fast`
- `goal_orientation`：`exam` / `application` / `exploration`

### 6.2 AgentResponse

```json
{
  "agent_name": "Navigator",
  "response_type": "knowledge",
  "content": { "message": "..." },
  "profile_update": {},
  "debate_report": {}
}
```

### 6.3 ResourcePackage

```json
{
  "concept": "变量与赋值",
  "document": "...",
  "mindmap": "...",
  "exercises": [...],
  "code_cases": [...],
  "audio_text": "..."
}
```

### 6.4 DebateReport

```json
{
  "status": "PASSED",
  "rounds": [
    { "round": 1, "agent": "Expert", "verdict": "PASS", "message": "...", "suggestion": "" }
  ],
  "final_votes": { "expert": "PASS", "teacher": "PASS", "student": "WARN", "guardian": "PASS" }
}
```

`status` 取值：`PASSED` / `MODIFIED` / `REJECTED`。  
`verdict` 取值：`PASS` / `WARN` / `REJECT`。

## 9. 维护约定

1. **新增接口**：必须在本文档中登记，并标注状态（✅/🚧/⏳）。
2. **新增字段**：优先 `snake_case`，字段缺失时返回 `null` 而非删除字段。
3. **废弃接口**：保留至少一个迭代，返回 `warning` 提示迁移。
4. **SSE 事件**：前端只解析 `data:` 行，不依赖 `event:` 字段。
5. **Mock 开发**：本地开发/CI 设置 `LLM_PROVIDER=mock`，避免真实 API Key 缺失导致接口 500。
