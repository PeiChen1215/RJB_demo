# 前后端接口对齐协议 v1.0

> 适用范围：智学蜂巢（EduHive）RJB Demo 项目
> 后端：FastAPI（`backend/app/api`）
> 前端：React + TypeScript + Axios（`frontend/src/services/api.ts`）
> 最后更新：2026-07-04

---

## 1. 目的

本文档统一前后端数据契约，明确：

- 接口前缀、认证方式、命名规范；
- 阻塞前端 Command Center 模块的关键字段与返回结构；
- 新增/改造接口（知识图谱后端驱动布局、BKT 默认参数标识、知识熔炉版本演进、Agent Trace 子阶段、资源生成 JSON body）的调用方式；
- 前端 `api.ts` 中方法与服务端路由的一一映射。

后端同学新增/修改接口前，应同步更新本文档；前端同学应以本文档为唯一字段依据，避免“试字段”。

## 2. 通用约定

### 2.1 Base URL

```text
开发环境：http://127.0.0.1:8000
API 前缀：/api
完整示例：http://127.0.0.1:8000/api/sessions/
```

前端 Axios 已配置 `baseURL: '/api'`（Vite dev proxy 会转发到 `http://127.0.0.1:8000/api`）。

### 2.2 命名规范

- **URL / 查询参数 / JSON 字段**：统一使用 `snake_case`。
- **前端 TypeScript 类型**：尽量与后端保持一致，也使用 `snake_case`，避免运行期转换。
- **路由路径**：小写、连字符分隔，如 `/chat-stream`、`/stream-generate`。

### 2.3 响应结构

本项目接口采用“直接返回业务对象”风格，**没有统一外层 `code / data / message` 包装**。

成功示例：

```json
{
  "session_id": "a1b2c3",
  "target_concept": "变量与赋值",
  "suggested_path": ["Python简介", "变量与赋值"]
}
```

失败示例：

```json
// 业务错误（200/400/404）
{ "error": "会话不存在" }

// FastAPI 校验错误（422）
{
  "detail": [
    { "loc": ["query", "session_id"], "msg": "field required", "type": "value_error.missing" }
  ]
}
```

### 2.4 认证

- 匿名会话：无需 Token，可直接调用 `/api/sessions/`、`/api/graph/*` 等。
- 用户会话：在请求头携带 `Authorization: Bearer <jwt_token>`。
- 当前实现中，除 `/api/sessions/` 列表页外，其余接口对匿名会话均兼容。

## 3. 接口清单

### 3.1 会话（Sessions）

| 方法 | 路径 | 说明 | 前端方法 |
|------|------|------|----------|
| POST | `/sessions/` | 创建会话 | `sessionApi.create(target_concept?)` |
| GET  | `/sessions/` | 列出当前用户会话（需登录） | `sessionApi.list()` |
| GET  | `/sessions/{session_id}` | 获取会话详情 | `sessionApi.get(session_id)` |
| GET  | `/sessions/{session_id}/profile` | 获取学生画像 | `sessionApi.getProfile(session_id)` |
| GET  | `/sessions/{session_id}/stats` | 学习行为统计 | `sessionApi.getStats(session_id)` |
| GET  | `/sessions/{session_id}/events` | 学习事件列表 | `sessionApi.getEvents(session_id)` |
| POST | `/sessions/{session_id}/events` | 记录学习事件 | `sessionApi.logEvent(session_id, ...)` |
| POST | `/sessions/{session_id}/behavior` | 前端行为埋点 | `sessionApi.logBehavior(session_id, ...)` |
| POST | `/sessions/{session_id}/chat` | 同步聊天 | `sessionApi.chat(session_id, ...)` |
| GET  | `/sessions/{session_id}/chat-stream` | SSE 流式聊天 | `sessionApi.chatStream(session_id, ...)` |
| POST | `/sessions/{session_id}/evaluate` | 触发评估 | `sessionApi.evaluate(session_id)` |
| GET  | `/sessions/{session_id}/agent-trace` | Agent 执行链路 | `sessionApi.getAgentTrace(session_id, limit?)` |

**创建会话请求体：**

```json
{
  "target_concept": "变量与赋值",
  "user_id": "anonymous"
}
```

**创建会话返回：**

```json
{
  "session_id": "uuid",
  "profile": { "knowledge_level": 1.0, ... },
  "target_concept": "变量与赋值",
  "suggested_path": ["Python简介", "变量与赋值"]
}
```

### 3.2 知识图谱（Graph）

| 方法 | 路径 | 说明 | 前端方法 |
|------|------|------|----------|
| GET | `/graph/` | 完整知识图谱（节点+边） | `graphApi.getGraph()` |
| GET | `/graph/layout` | 后端驱动布局（含坐标/颜色） | `graphApi.getLayout()` |
| GET | `/graph/path` | 学习路径/个人路径高亮 | `graphApi.getPath(...) / getPathWithProfile(...)` |
| GET | `/graph/concept/{name}` | 知识点详情 | `graphApi.getConcept(name)` |

#### 3.2.1 `/graph/layout` 返回字段（阻塞 C 前端）

```json
{
  "nodes": [
    {
      "id": "变量与赋值",
      "name": "变量与赋值",
      "module": "基础语法",
      "difficulty": 1,
      "x": 220,
      "y": 0,
      "color": "#3b82f6"
    }
  ],
  "edges": [
    { "source": "Python简介", "target": "变量与赋值", "strength": 0.8 }
  ]
}
```

约定：

- `x`、`y` 为后端基于拓扑层级计算的坐标，前端直接渲染，不再自行布局。
- `color` 按模块分配，同模块节点颜色一致。
- `difficulty` 取值 1-5。

#### 3.2.2 `/graph/path` 调用方式

支持三种调用模式：

1. **旧方式（纯路径）**

   ```text
   GET /api/graph/path?from_concepts=Python简介,基本数据类型&to_concept=变量与赋值
   ```

   返回：`{ "from": [...], "to": "...", "path": [...] }`

2. **个人路径高亮（session 驱动）**

   ```text
   GET /api/graph/path?session_id=<uuid>&target_concept=变量与赋值
   ```

   `target_concept` 可选，不传则使用会话当前目标。

   返回：

   ```json
   {
     "session_id": "uuid",
     "target_concept": "变量与赋值",
     "path_nodes": [
       {
         "id": "Python简介",
         "name": "Python简介",
         "mastery_probability": 0.95,
         "is_mastered": true,
         "is_current": false,
         "state": "mastered"
       },
       {
         "id": "变量与赋值",
         "name": "变量与赋值",
         "mastery_probability": 0.42,
         "is_mastered": false,
         "is_current": true,
         "state": "current"
       }
     ],
     "path_edges": [
       {
         "source": "Python简介",
         "target": "变量与赋值",
         "reason": "...",
         "prerequisites": [...],
         "pitfalls": [...]
       }
     ],
     "mastered_concepts": ["Python简介"]
   }
   ```

   `state` 枚举：`mastered` / `current` / `waiting`。

3. **空参数**：返回 `{ "error": "请提供 from_concepts + to_concept，或 session_id + target_concept" }`，HTTP 200，前端应友好提示。

### 3.3 学习效果评估（Evaluation）

| 方法 | 路径 | 说明 | 前端方法 |
|------|------|------|----------|
| GET | `/evaluation/heatmap` | 掌握度热力图 | `evaluationApi.getHeatmap(session_id)` |
| GET | `/evaluation/bkt` | BKT 模型状态 | `evaluationApi.getBKT(session_id, concept?)` |
| POST | `/evaluation/analyze` | 触发完整分析 | `evaluationApi.analyze(session_id)` |

#### 3.3.1 `/evaluation/bkt` 新增字段（阻塞 C 前端）

单知识点查询返回：

```json
{
  "session_id": "uuid",
  "concept": "变量与赋值",
  "mastery_probability": 0.42,
  "sample_count": 0,
  "is_default": true,
  "last_updated": "2026-07-04T00:00:00+08:00",
  "explanation": "默认参数，练习数据不足",
  "bkt_params": {
    "p_init": 0.2,
    "p_transit": 0.1,
    "p_slip": 0.1,
    "p_guess": 0.2,
    "observation_count": 0
  }
}
```

约定：

- `is_default = true` 表示该知识点尚无练习记录，前端可提示“默认参数，仅供参考”。
- `sample_count` 为观察次数。
- `explanation` 为人类可读说明。

### 3.4 资源生成（Resources）

| 方法 | 路径 | 说明 | 前端方法 |
|------|------|------|----------|
| POST | `/resources/generate` | 同步生成（新 JSON body） | `resourceApi.generate(concept, profile?)` |
| POST | `/resources/generate-for-session/{session_id}` | 为指定会话生成 | `resourceApi.generateForSession(...)` |
| GET  | `/resources/stream-generate` | SSE 流式生成 | `resourceApi.generateStream(sessionId, concept)` |
| GET  | `/resources/versions` | 知识点版本演进 | `resourceApi.getVersions(concept)` |
| GET  | `/resources/latest` | 最新资源 | `resourceApi.getLatest(concept)` |
| GET  | `/resources/evolution` | 版本差异 + 错误率 | `resourceApi.getEvolution(concept)` |
| GET  | `/resources/thinking-path` | 生成过程回放 | `resourceApi.getThinkingPath(concept)` |

#### 3.4.1 `/resources/generate` JSON Body 约定（阻塞 C 前端）

兼容旧 query 参数，同时支持 JSON body：

```json
{
  "concept": "变量与赋值",
  "session_id": "uuid",
  "profile": {
    "knowledge_level": 2,
    "cognitive_modality": "visual"
  }
}
```

- `concept`：必填。
- `session_id`：可选。若提供，则复用/创建该会话；否则创建 `default` 临时会话。
- `profile`：可选。若提供，会合并到会话画像中用于本次生成，但不会持久化为新画像。

返回：与旧接口一致，为 `orchestrator.generate_resource()` 结果，含 `concept`、`package`、`debate_report`。

#### 3.4.2 `/resources/versions` 返回字段（阻塞 C 前端）

```json
{
  "concept": "变量与赋值",
  "versions": [
    {
      "version": 1,
      "resource_id": "uuid",
      "created_at": "2026-07-04T00:00:00",
      "triggered_by": "initial",
      "change_reason": "首次生成",
      "status": "approved",
      "content_snapshot": {
        "document": "# 变量与赋值...",
        "exercises": [...],
        "code_cases": [...]
      }
    }
  ]
}
```

约定：

- `triggered_by` 取值：`initial`、`error_rate`、`manual`、`revision` 等。
- `status` 取值：`approved` / `rejected`。
- `content_snapshot` 为版本快照，可能较大，前端按需展示。

### 3.5 Agent Trace

| 方法 | 路径 | 说明 | 前端方法 |
|------|------|------|----------|
| GET | `/sessions/{session_id}/agent-trace?limit=20` | 获取 Agent 链路 | `sessionApi.getAgentTrace(session_id, limit?)` |

Trace 单条字段：

```json
{
  "trace_id": "uuid",
  "agent_name": "Reviewer/Debate",
  "stage": "debate",
  "intent": "资源生成",
  "status": "success",
  "started_at": "2026-07-04T00:00:00",
  "finished_at": "2026-07-04T00:00:01",
  "duration_ms": 120,
  "error_message": ""
}
```

约定：

- `agent_name` 已扩展子阶段，如 `Reviewer/Debate`、`Reviewer/Guardian`。
- 前端 AgentPanel 轮询时，按 `started_at` 排序展示即可。

## 6. 注意事项

1. **字段缺失时**：后端宁可返回 `null` 也不删除字段，避免前端 `undefined` 报错。
2. **新增字段时**：优先使用 `snake_case`，并在本文档“关键字段速查”中登记。
3. **接口废弃时**：保留旧接口至少一个迭代，返回 `warning` 字段提示迁移，不要直接删除。
4. **SSE 接口**：前端使用原生 `EventSource` 或 `fetch` 读取，事件类型只解析 `data:` 行，不依赖 `event:` 字段。
5. **Mock 模式**：开发/测试时设置环境变量 `LLM_PROVIDER=mock`，避免真实 API Key 缺失导致接口 500。
