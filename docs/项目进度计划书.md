## 一、项目目标

在 4 周内完成一个可运行、可演示、具备国一竞争力的 **Python 个性化学习多智能体系统**。系统以大模型为主推理引擎，以知识图谱为神经符号约束基础，实现对话式画像构建、多 Agent 协同资源生成、辩论议会质量审核、认知风格自适应渲染等核心功能。

## 十、实际开发偏差与复盘

### 10.1 与原计划不一致的地方

| 原计划 | 实际情况 | 原因 |
|--------|---------|------|
| 讯飞星火 4.5 Max / Spark Pro | 使用 DeepSeek API | 本地调试更方便，接口兼容 OpenAI SDK |
| 第 1 周完成 Docker 环境 | Docker 未安装，使用 MemoryGraph fallback | 当前机器无 Docker，为不阻塞开发先降级 |
| 第 3 周中后期做 UI 美化 | 第 2 周末即开始全面重构 | Tailwind 样式失效、TS 构建报错集中爆发，必须提前修 |
| HTTPS 推送 GitHub | 改为 SSH 推送 | 当前环境 HTTPS 连 GitHub 被重置，SSH 可用 |
| 测试文件使用占位 key | 测试文件曾硬编码真实 key | 初期为了本地跑通测试，后通过 GitHub Secret Scanning 发现并清理 |

### 10.2 关键踩坑与解决方案

1. **Tailwind 样式只有 4KB，组件无样式**
   - 根因：`frontend/postcss.config.js` 缺失，Tailwind 未被 PostCSS 处理。
   - 解决：补回 `postcss.config.js`（`tailwindcss` + `autoprefixer`），CSS 恢复到 62KB。

2. **`npm run build` TypeScript 报错**
   - 根因：未使用的导入（`Button`、`Skeleton`、`IconBox`）和 `framer-motion` 的 `onDrag` 与原生 DOM 事件类型冲突。
   - 解决：清理未使用导入；调整 `glass-card.tsx` 不直接透传 drag 事件。

3. **Windows 路径反斜杠导致 AST 安全校验误报**
   - 根因：代码安全过滤器把 Windows 路径中的 `\` 当成转义字符处理。
   - 解决：在过滤逻辑中对 Windows 路径做归一化/白名单处理。

4. **GitHub 推送被 Secret Scanning 拦截**
   - 根因：`backend/test_*.py` 中硬编码了 DeepSeek API Key。
   - 解决：
     - 将 5 个测试文件中的真实 key 改为 `os.getenv("DEEPSEEK_API_KEY", "sk-PLACEHOLDER")`。
     - 使用 `git rebase -i --root` 改写第一个提交的历史，确保所有提交中不再包含真实 key。
     - 通过 `git log --all -S <key>` 验证无残留后，SSH 强制推送成功。

5. **HTTPS 无法连接 GitHub**
   - 根因：当前网络环境访问 `github.com:443` 被重置。
   - 解决：改用 SSH（`git@github.com:PeiChen1215/RJB_demo.git`）推送，并将 `origin` 远程地址同步切换为 SSH。

### 10.4 经验沉淀

- **Git 安全**：永远不要把真实 API Key 写进代码，即使是测试文件；使用 `.env` + `.gitignore` + GitHub Secret Scanning。
- **前端工程化**：创建 Vite + Tailwind 项目时，第一时间确认 `postcss.config.js` 存在。
- **网络推送**：遇到 HTTPS 被重置时，优先尝试 SSH；同时准备好改写历史的能力。
- **防御式降级**：Docker / Neo4j 不可用时，MemoryGraph fallback 保证了开发不中断。
